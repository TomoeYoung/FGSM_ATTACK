# FGSM Attack on MNIST

Учебный проект на **PyTorch**, который показывает работу adversarial-атаки **FGSM** (`Fast Gradient Sign Method`) на модели для распознавания рукописных цифр MNIST.

Скрипт берёт случайное изображение цифры, пропускает его через нейросеть, считает градиент ошибки по входной картинке и добавляет к пикселям небольшое направленное возмущение. В результате изображение для человека почти не меняется, но модель может начать ошибаться.

## Состав проекта

```text
.
├── backdoored_model.pt   # сохранённые веса модели PyTorch
├── fgsm_attack.py        # основной скрипт с FGSM-атакой
└── README.md             # описание проекта
```

## Как запустить

Установить зависимости:

```bash
pip install torch torchvision matplotlib
```

Запустить скрипт:

```bash
python fgsm_attack.py
```

При первом запуске датасет **MNIST** будет автоматически скачан в папку `data`.

## Что делает `fgsm_attack.py`

Скрипт выполняет следующие шаги:

1. Создаёт архитектуру полносвязной нейросети `FCNet`.
2. Загружает веса из файла `backdoored_model.pt`.
3. Загружает тестовый набор MNIST.
4. Берёт случайное изображение цифры.
5. Делает предсказание модели до атаки.
6. Считает функцию потерь `cross_entropy`.
7. Считает градиент ошибки по входному изображению.
8. Создаёт adversarial-изображение с помощью FGSM.
9. Делает предсказание после атаки.
10. Показывает визуализацию: оригинал, градиент, знак градиента, возмущение и итоговое изображение.

## Архитектура модели

В проекте используется простая полносвязная сеть:

```python
self.net = nn.Sequential(
    nn.Linear(784, 512), nn.ReLU(),
    nn.Linear(512, 256), nn.ReLU(),
    nn.Linear(256, 128), nn.ReLU(),
    nn.Linear(128, 10)
)
```

MNIST-картинка имеет размер `28 x 28` пикселей. Так как модель состоит из `Linear`-слоёв, изображение перед подачей в сеть разворачивается в один вектор.

Для одной картинки:

$$
X \in \mathbb{R}^{1 \times 28 \times 28}
$$

После разворачивания:

$$
x \in \mathbb{R}^{784}
$$

Потому что:

$$
28 \cdot 28 = 784
$$

Для batch из нескольких картинок форма выглядит так:

$$
X \in \mathbb{R}^{B \times 1 \times 28 \times 28}
$$

После `view(-1, 784)`:

$$
x \in \mathbb{R}^{B \times 784}
$$

В коде это делается здесь:

```python
def forward(self, x):
    return self.net(x.view(-1, 784))
```

То есть матрица изображения:

$$
\begin{bmatrix}
p_1 & p_2 & p_3 & \dots & p_{28} \\
p_{29} & p_{30} & p_{31} & \dots & p_{56} \\
\vdots & \vdots & \vdots & \ddots & \vdots \\
p_{757} & p_{758} & p_{759} & \dots & p_{784}
\end{bmatrix}
$$

разворачивается в вектор:

$$
\begin{bmatrix}
p_1 & p_2 & p_3 & \dots & p_{784}
\end{bmatrix}
$$

## Как данные проходят через `Linear`-слои

Каждый слой `nn.Linear` делает матричное преобразование:

$$
y = xW^T + b
$$

В PyTorch для слоя:

```python
nn.Linear(in_features, out_features)
```

матрица весов хранится в форме:

$$
W \in \mathbb{R}^{out\_features \times in\_features}
$$

Например, для первого слоя:

```python
nn.Linear(784, 512)
```

формы такие:

$$
x \in \mathbb{R}^{B \times 784}
$$

$$
W \in \mathbb{R}^{512 \times 784}
$$

$$
b \in \mathbb{R}^{512}
$$

Чтобы перемножение было возможно, PyTorch использует транспонированную матрицу весов:

$$
xW^T + b
$$

По размерностям это выглядит так:

$$
(B \times 784) \cdot (784 \times 512) + (512) = B \times 512
$$

После этого применяется `ReLU`:

$$
\mathrm{ReLU}(z) = \max(0, z)
$$

Полный проход по размерам:

```text
Input image:      [B, 1, 28, 28]
Flatten:          [B, 784]
Linear 784->512:  [B, 512]
ReLU:             [B, 512]
Linear 512->256:  [B, 256]
ReLU:             [B, 256]
Linear 256->128:  [B, 128]
ReLU:             [B, 128]
Linear 128->10:   [B, 10]
```

Последние `10` чисел — это **logits**, то есть сырые оценки модели для классов от `0` до `9`.

Если обозначить выход модели как `z`, то:

$$
z = f_\theta(x)
$$

где:

- $x$ — входная картинка, развёрнутая в вектор;
- $\theta$ — веса модели;
- $z \in \mathbb{R}^{10}$ — logits для десяти цифр.

## Softmax и вероятность класса

Чтобы превратить logits в вероятности, используется `softmax`:

$$
p_i = \frac{e^{z_i}}{\sum_{j=0}^{9} e^{z_j}}
$$

где:

- $z_i$ — logit для класса $i$;
- $p_i$ — вероятность класса $i$.

В коде:

```python
probs = F.softmax(logits, dim=1)
```

Предсказанный класс выбирается как индекс максимального logit:

```python
pred_before = logits.argmax(dim=1).item()
```

Математически:

$$
\hat{y} = \arg\max_i z_i
$$

## Функция потерь

Для классификации используется `cross entropy`:

```python
loss = F.cross_entropy(logits, y)
```

Если правильный класс — $y$, то loss можно записать так:

$$
L(x, y) = -\log p_y
$$

Смысл простой: если модель дала маленькую вероятность правильному классу, значение ошибки будет большим.

## Как считается градиент по изображению

Обычно при обучении считают градиенты по весам модели. В FGSM нужно другое: посчитать градиент ошибки по входной картинке.

В коде это начинается здесь:

```python
x_adv = x.clone().detach().requires_grad_(True)
```

`requires_grad_(True)` говорит PyTorch, что для этого тензора нужно отслеживать операции и потом посчитать производную loss по нему.

Дальше выполняется обычный forward pass:

```python
logits = model(x_adv)
loss = F.cross_entropy(logits, y)
```

После этого вызывается обратное распространение ошибки:

```python
loss.backward()
```

PyTorch считает градиент:

$$
\nabla_x L(x, y) = \frac{\partial L}{\partial x}
$$

То есть для каждого пикселя определяется, как изменение этого пикселя повлияет на ошибку модели.

Форма градиента совпадает с формой входного изображения:

$$
x \in \mathbb{R}^{B \times 1 \times 28 \times 28}
$$

$$
\nabla_x L \in \mathbb{R}^{B \times 1 \times 28 \times 28}
$$

Поэтому градиент можно показать как обычную картинку размером `28 x 28`.

## Как градиент проходит назад через матрицы

Рассмотрим один линейный слой:

$$
y = xW^T + b
$$

Пусть на обратном проходе к этому слою пришёл градиент:

$$
\frac{\partial L}{\partial y}
$$

Тогда градиент по входу слоя считается так:

$$
\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} W
$$

По размерностям:

$$
\frac{\partial L}{\partial y} \in \mathbb{R}^{B \times out\_features}
$$

$$
W \in \mathbb{R}^{out\_features \times in\_features}
$$

Тогда:

$$
(B \times out\_features) \cdot (out\_features \times in\_features) = B \times in\_features
$$

Например, для последнего слоя `Linear(128, 10)`:

$$
\frac{\partial L}{\partial y} \in \mathbb{R}^{B \times 10}
$$

$$
W \in \mathbb{R}^{10 \times 128}
$$

Значит градиент по входу этого слоя будет:

$$
\frac{\partial L}{\partial x} \in \mathbb{R}^{B \times 128}
$$

То есть обратный проход постепенно идёт от выхода модели обратно к входу:

```text
[B, 10]
   ↓ Linear 128->10 backward
[B, 128]
   ↓ ReLU backward
[B, 128]
   ↓ Linear 256->128 backward
[B, 256]
   ↓ ReLU backward
[B, 256]
   ↓ Linear 512->256 backward
[B, 512]
   ↓ ReLU backward
[B, 512]
   ↓ Linear 784->512 backward
[B, 784]
   ↓ reshape back
[B, 1, 28, 28]
```

Именно это можно назвать «сворачиванием» градиента обратно к форме изображения: сначала градиент имеет форму выхода модели, а после цепочки обратных преобразований снова становится тензором пикселей.

## Обратный проход через ReLU

`ReLU` работает так:

$$
\mathrm{ReLU}(x) = \max(0, x)
$$

Её производная:

$$
\mathrm{ReLU}'(x) =
\begin{cases}
1, & x > 0 \\
0, & x \le 0
\end{cases}
$$

Значит при обратном проходе градиент проходит только через те нейроны, которые были активны на forward pass.

Если значение до `ReLU` было положительным, градиент проходит дальше. Если значение было отрицательным или нулевым, градиент обнуляется.

## Правило цепочки

Вся нейросеть — это композиция функций:

$$
f(x) = f_4(f_3(f_2(f_1(x))))
$$

Поэтому градиент считается по правилу цепочки:

```math
\frac{\partial L}{\partial x} = \frac{\partial L}{\partial f_4} \frac{\partial f_4}{\partial f_3} \frac{\partial f_3}{\partial f_2} \frac{\partial f_2}{\partial f_1} \frac{\partial f_1}{\partial x}
```

Вручную это считать неудобно, поэтому PyTorch строит граф вычислений и автоматически делает backpropagation после вызова:

```python
loss.backward()
```

## Формула FGSM

FGSM создаёт атакованное изображение так:

$$
x_{adv} = x + \varepsilon \cdot \mathrm{sign}(\nabla_x L(x, y))
$$

где:

- $x$ — исходное изображение;
- $x_{adv}$ — adversarial-изображение;
- $\varepsilon$ — сила атаки;
- $L(x, y)$ — функция потерь;
- $\nabla_x L(x, y)$ — градиент ошибки по изображению;
- $\mathrm{sign}(\nabla_x L)$ — знак градиента.

В коде:

```python
grad = x_adv.grad.detach()
signed_grad = grad.sign()

x_adv = x_adv + epsilon * signed_grad
x_adv = torch.clamp(x_adv, 0, 1).detach()
```

`grad.sign()` превращает каждый элемент градиента в одно из трёх значений:

$$
\mathrm{sign}(g_i) =
\begin{cases}
1, & g_i > 0 \\
0, & g_i = 0 \\
-1, & g_i < 0
\end{cases}
$$

То есть каждый пиксель либо немного увеличивается, либо немного уменьшается, либо остаётся без изменений.

## Визуализация

После запуска появляется окно с пятью изображениями:

```text
Original        — исходная цифра
Gradient dL/dx  — градиент ошибки по пикселям
sign(gradient)  — направление изменения пикселей
Perturbation    — добавленное возмущение
Adversarial     — итоговая картинка после атаки
```

Основная идея визуализации: показать не только итоговую атакованную картинку, но и то, за счёт чего она получилась.
